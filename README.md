## 🖥️ What ReqPipeline Does

ReqPipeline helps you review and shape software requirements with support from a local LLM. It brings together USDM ideas, plain requirement checks, and a simple Windows app so you can work through a requirements set in one place.

Use it to:
- review requirement text
- spot unclear wording
- check for gaps and conflicts
- support design discussions
- keep notes in one flow

It is built for people who want help with requirements without setting up a complex tool stack.

## ⚙️ Before You Start

ReqPipeline is designed for Windows desktops and works best on a machine that can run a local model tool such as Ollama.

Suggested setup:
- Windows 10 or Windows 11
- 8 GB RAM or more
- 5 GB free disk space
- Internet access for the first download
- Ollama installed if you want local LLM support

A stronger PC will give you smoother response times when you use model-based review features.

## 🤖 Local LLM Setup

ReqPipeline can use a local language model for review tasks. A local model runs on your own computer, which keeps your work in one place.

Basic setup steps:
1. Install Ollama if you do not have it yet.
2. Start Ollama on your PC.
3. Download a model in Ollama.
4. Open ReqPipeline.
5. Point the app to your local Ollama service if needed.
6. Run a test review on a short requirement.

A good first model is a small one that runs well on a normal desktop. If your PC has more RAM, you can use a larger model.

## 🧭 First Run

When you open ReqPipeline for the first time, you may see a clean workspace with sections for requirements, review results, and pipeline steps.

Try this first:
1. Add one requirement line.
2. Mark the source or project name.
3. Run a review pass.
4. Read the feedback.
5. Edit the requirement if the wording is unclear.
6. Run the review again.

This helps you see how the app fits into your work before you load a full set of requirements.

## 📝 Typical Work Flow

ReqPipeline follows a simple review flow:

1. Paste or enter requirements.
2. Group them by project or feature.
3. Review them for clarity and completeness.
4. Use the LLM help to spot weak wording.
5. Check the output against USDM-style thinking.
6. Refine the text.
7. Save the updated set.

This makes it easier to move from rough notes to cleaner requirements.

## 🛠️ Troubleshooting

### The app does not open
- Check that the file finished downloading.
- If the file is in a ZIP folder, extract it first.
- Right-click the app and choose Run as administrator.
- Make sure Windows did not block the file.

### Ollama is not found
- Confirm Ollama is installed.
- Make sure Ollama is running.
- Check that the local service is available on your machine.
- Try restarting Ollama and ReqPipeline.

### Reviews feel slow
- Use a smaller local model.
- Close other heavy apps.
- Check your free RAM.
- Restart the model service if needed.

### Text does not look right
- Reopen the source requirement.
- Check for pasted formatting from another app.
- Try plain text input.
- Use short requirement lines first.

## 📚 Good Input Format

ReqPipeline works best when your requirements are simple and direct.

Example:
- The system shall allow users to reset a password.
- The app shall save the report as a PDF.
- The tool shall flag unclear requirement text.

Good input:
- one idea per line
- one requirement per sentence
- clear subject and action
- no long paragraphs

This helps the review step give cleaner feedback.

## 🧪 Example Review Flow

1. Write a requirement:
   - The system shall let the user export data.

2. Run review:
   - The wording may be too broad.
   - The export format is not named.
   - The user role is not defined.

3. Improve it:
   - The system shall let a signed-in user export project data as CSV.

This kind of edit is the core value of the pipeline.
