# レビューの評価基準
以下の観点でツリー全体を分析し、問題点を抽出してください。一般論での指摘は禁止です。必ず対象ノードの具体的な文章を引用して、論理的に説明してください。

1. **SpecificationConflict (仕様間の衝突・矛盾)** 【最重要】
   同じツリー内に存在する複数の「[仕様]」間で、同じ条件（トリガー）に対する振る舞いが矛盾・衝突していませんか？（例：エラー時にロックする仕様と、入力を無視する仕様が混在している等）
2. **IntentMismatch (目的と手段の不一致)**
   親ノードの「[理由]」が示唆する目的・方向性と、子ノードの「[仕様]」の振る舞いが、意味的に逆行または乖離していませんか？
3. **GuidelineViolation (ガイドライン違反)**
   【評価ガイドライン】に記載されている禁止事項や評価基準に違反していませんか？

# 出力形式（Output Format）【超厳格】
- 出力は **絶対にJSON配列のみ** にしてください。
- Markdownのコードブロック（```json など）や、前置き・挨拶・考察などのテキストは一切出力してはいけません。最初の文字は `[` 、最後の文字は `]` です。
- 矛盾や不一致がない場合は空の配列 `[]` を出力してください。
- 以下のJSONスキーマに厳密に従ってください。

[
  {
    "NodeId": "該当するノードのID（必須）",
    "RuleId": "SpecificationConflict, IntentMismatch, または AI-REVIEW-001",
    "Message": "【対象の文章の引用】＋【なぜ矛盾・不一致しているのかの論理的な説明】",
    "Severity": "Error" または "Warning"
  }
]
